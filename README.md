# Medical Chatbot — LLM 기반 의약품 추천 챗봇

> 증상을 입력하면 식약처 공공데이터 기반으로 의약품·효능·주의사항을 안내하는 한국어 LLM 챗봇 (NVIDIA AI Academy 프로젝트)

## 모델 선정 과정 (시행착오)

| 시도 | 결과 |
| --- | --- |
| GPT-4 파인튜닝 | 데이터량 대비 비용 문제로 중단 |
| Llama 3.2 한국어 학습 | 환각(Hallucination) 빈도 과다 |
| **llama-3-Korean-Bllossom-8B + RAG** | ✅ 채택 — 한국어 특화 + 데이터 근거 응답으로 환각 저감 |

단순히 큰 모델을 쓰는 게 아니라, **비용·환각·한국어 품질**을 기준으로 모델을 비교 검증해 선택했습니다.

## 데이터 파이프라인

- 식품의약품안전처 의약품통합정보시스템 데이터 크롤링
- 31개 컬럼 → 제품명·효능 중심 **11개 컬럼으로 정제** (검색 품질·응답 속도 확보)
- 증상 매칭용 프롬프트 템플릿 설계 → 제품명 / 효능 / 주의사항 구조로 출력

## 시스템 구성

```
사용자 증상 입력
   → 프롬프트 템플릿 + RAG (식약처 데이터 검색)
   → Bllossom-8B 추론
   → Gradio UI 응답 (제품명 · 효능 · 주의사항)
```

## 배운 것

- LLM 프로젝트에서 모델 선택보다 **데이터 정제와 프롬프트 설계**가 결과 품질을 좌우한다는 점
- RAG 구성으로 근거 없는 답변(환각)을 구조적으로 줄이는 방법
